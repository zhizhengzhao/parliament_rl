# 2026_4_16_v1

## Changes from v3
- **React-advance flow**: Restructured "How to Contribute" into three phases: react (vote + comment), advance (post), submit. Vote and comment are peers, not hierarchical.
- **Post as output**: "Substantive progress must live in posts — the secretary compiles posts as the parliament's output." Prevents discussion from living only in comments.
- **Focused posts**: "Each post should be a focused, verifiable logical step." Discourages long multi-step derivations.
- **Post cross-referencing**: "Connect your work to the existing discussion (Building on P_3, ...)." Reduces self-contained posting.
- **Judge as guide**: "Your mission is to guide the parliament toward the correct answer." Frames judge role as steering, not just grading.
- **No re-voting**: "Vote only on new posts and comments you haven't voted on yet."
- **python_exec output truncation**: 10,000 char limit prevents context window overflow.
- **python_exec softened**: "Not mandatory if you are fully confident" for both actors and judges.
- **Dead code cleanup**: removed follows table, unused _exec method, stale exit reason.
